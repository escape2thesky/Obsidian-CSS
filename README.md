# Document Editor - Obsidian Plugin 操作説明書

技術論文・報告書の作成を支援するObsidianプラグインです。
見出し自動番号付け・図表管理・相互参照・数式番号・目次生成・PDF出力を提供します。

---

## インストール方法

### ステップ 1: ビルド

```bash
cd document-editor
npm install   # 初回のみ
npm run build
```

ビルドが完了すると `document-editor/` フォルダ内に以下が生成されます。

### ステップ 2: 必要な3ファイルをプラグインフォルダにコピー

Obsidianプラグインとして動作するのに必要なファイルは以下の **3つだけ** です。
`src/`・`node_modules/`・`package.json` 等の開発ファイルはコピー不要です。

| ファイル | 説明 |
|---------|------|
| `main.js` | ビルド済みプラグイン本体 |
| `manifest.json` | プラグイン情報（Obsidianが読み込む） |
| `styles.css` | プラグインのスタイル定義 |

コピー先: **`<VaultのRoot>/.obsidian/plugins/document-editor/`**

```
<Vault>/
└── .obsidian/
    └── plugins/
        └── document-editor/   ← このフォルダを作成してコピー
            ├── main.js
            ├── manifest.json
            └── styles.css
```

### ステップ 3: プラグインを有効化

1. Obsidianを再起動（または Ctrl+R でリロード）
2. 設定 → コミュニティプラグイン → 「Document Editor」を **有効化**

## 機能一覧と使い方

### 1. 見出し自動番号付け

見出しに自動で階層番号を表示します（**Markdownファイルは変更されません**）。

```markdown
# はじめに          → 表示: 1. はじめに
## 背景             → 表示: 1.1. 背景
## 目的             → 表示: 1.2. 目的
# 方法              → 表示: 2. 方法
### 実験手順        → 表示: 2.1.1. 実験手順
```

- Live Preview・Reading View の両方で番号が表示されます
- 番号はH1〜H6まで対応（例: `1.1.1.1.`）
- 設定でON/OFFを切り替えられます

---

### 2. wiki画像リンクの自動変換

スクリーンショットや画像をペーストすると、Obsidianは `![[image.png]]` 形式（wiki リンク）で挿入します。
本プラグインはこれを自動で標準Markdown形式に変換します。

```
ペースト直後:  ![[Pasted image 2024-01-01.png]]
       ↓ 自動変換（300ms後）
変換後:        ![Pasted image 2024-01-01](Pasted image 2024-01-01.png)
```

エイリアス付きの場合:

```
![[fig1.png|実験結果]]  →  ![実験結果](fig1.png)
```

- 設定でON/OFFを切り替えられます
- 変換はUndo（Ctrl+Z）で戻せます

---

### 3. 図の自動番号付け & 中央揃え

画像を挿入すると、自動で中央揃えになり「図N: キャプション」が表示されます。

```markdown
![グラフの説明](graph.png)
```

**表示結果:**
> [グラフの説明（中央揃え）]
> 図1: グラフの説明

- キャプションは `![]()` の `[]` 内に書いたテキストが使われます
- 設定でON/OFFを切り替えられます

---

### 4. 表の自動番号付け & 中央揃え

テーブルを書くと自動で中央揃えになり「表N」が上部に表示されます。

```markdown
| 列1 | 列2 |
|-----|-----|
| A   | B   |
```

**表示結果:**
> 表1
> [テーブル（中央揃え）]

---

### 5. ラベルと相互参照

TeXのように要素にラベルを付け、文書内のどこからでも参照できます。

#### ラベルの付け方

```markdown
![実験結果](result.png){#fig:experiment}

| 比較表 |
|--------|
| データ |
{#tbl:comparison}

# はじめに {#sec:intro}

$$
E = mc^2
$$
{#eq:einstein}
```

#### 参照の書き方

```markdown
[ref:fig:experiment]   → 表示: 図1（クリックで図にジャンプ）
[ref:tbl:comparison]   → 表示: 表2（クリックで表にジャンプ）
[ref:sec:intro]        → 表示: 1.（クリックで見出しにジャンプ）
[ref:eq:einstein]      → 表示: (1)（クリックで数式にジャンプ）
[ref:bib:smith2020]    → 表示: [1]（クリックで参考文献にジャンプ）
```

**ラベルが存在しない場合:** `[???]` または `[?]` と表示されます

#### ラベルの種類

| プレフィックス | 対象 | 例 |
|--------------|------|-----|
| `fig:` | 図 | `{#fig:chart1}` |
| `tbl:` | 表 | `{#tbl:result}` |
| `eq:` | 数式 | `{#eq:maxwell}` |
| `sec:` | 見出し | `{#sec:intro}` |

---

### 6. 数式の自動番号付け

ブロック数式（`$$...$$`）に自動で式番号が付きます。

```markdown
$$
E = mc^2
$$
{#eq:einstein}
```

**表示結果:**
> E = mc²　　　　　　　　　　　　　(1)

- ラベル（`{#eq:...}`）は数式ブロックの**直後の行**に書きます
- 参照: `式 [ref:eq:einstein] より...` → `式 (1) より...`

---

### 7. 目次の自動生成

コマンドパレット（`Ctrl+P` / `Cmd+P`）から実行できます。

| コマンド | 動作 |
|---------|------|
| `Document Editor: TOCをファイル先頭に挿入` | ファイルの一番上に目次を挿入 |
| `Document Editor: TOCをカーソル位置に挿入` | カーソルのある位置に目次を挿入 |

**生成される目次の例:**

```markdown
## 目次
- 1. はじめに
  - 1.1. 背景
  - 1.2. 目的
- 2. 方法
  - 2.1. 実験手順
- 3. 結果
```

---

### 8. 参考文献管理

`key` カラムを持つMarkdownテーブルが参考文献として自動認識されます。

#### 参考文献テーブルの書き方

```markdown
| key | 著者 | タイトル | 雑誌 | 年 |
|-----|------|----------|------|-----|
| smith2020 | Smith et al. | Deep Learning for NLP | Nature | 2020 |
| jones2021 | Jones, A. | Neural Networks Review | Science | 2021 |
```

> **必須:** `key` という名前のカラムを含めてください。

#### 参照の書き方

```markdown
深層学習の研究 [ref:bib:smith2020] によると...
```

**表示結果:**
> 深層学習の研究 [1] によると...（クリックで参考文献テーブルへジャンプ）

#### 番号の付け方（設定で切替可能）

- **参照順（デフォルト）**: 文書内で最初に参照された順番に番号を付与
- **テーブル行順**: テーブルの上から順番に番号を付与

---

### 9. PDF出力

コマンドパレットから `Document Editor: PDFを出力` を実行します。

1. **設定ダイアログが開きます:**
   - 本文フォントサイズ（9〜14pt）
   - 見出しフォントサイズ比率
   - 余白（mm）
   - 出力ファイル名

2. **右側にA4プレビューが表示されます:**
   - フォントサイズを変更すると自動でプレビューが更新されます
   - ページ区切りの目安線が表示されます

3. **「PDFを生成」ボタンをクリック:**
   - VaultフォルダにPDFファイルが保存されます
   - 見出し番号・図表番号・数式番号・参考文献番号が反映されます

> **注意:** PDF出力はObsidian Desktop版専用です（iOS/Android非対応）

## 設定項目

Obsidian設定 > **Document Editor** から変更できます。

| 設定項目 | 説明 | デフォルト |
|---------|------|-----------|
| 見出し自動番号付け | 見出しに番号を表示するか | ON |
| wiki画像リンクを自動変換 | `![[...]]` を `![](...)` に自動変換するか | ON |
| 図番号を自動付与 | 画像に図番号を表示するか | ON |
| 表番号を自動付与 | テーブルに表番号を表示するか | ON |
| 数式番号を自動付与 | ブロック数式に番号を付けるか | ON |
| 参考文献管理 | 参考文献テーブルを管理するか | ON |
| 参考文献の番号順 | 参照順 / テーブル行順 | 参照順 |
| デフォルト本文フォントサイズ | PDF出力時のデフォルト | 11pt |
| デフォルト余白 | PDF出力時のデフォルト余白 | 20mm |

## 構文リファレンス

### ラベル構文

```
{#fig:ラベル名}    図のラベル（画像の後に記載）
{#tbl:ラベル名}    表のラベル（テーブルの後に記載）
{#eq:ラベル名}     数式のラベル（数式ブロック直後の行に記載）
{#sec:ラベル名}    見出しのラベル（見出し行末に記載）
```

### 参照構文

```
[ref:fig:ラベル名]    図を参照
[ref:tbl:ラベル名]    表を参照
[ref:eq:ラベル名]     数式を参照
[ref:sec:ラベル名]    見出しを参照
[ref:bib:キー名]      参考文献を参照
```

### 参考文献テーブル

```markdown
| key | 著者 | タイトル | 年 |
|-----|------|----------|-----|
| キー名 | ... | ... | ... |
```

**必須:** `key` という名前のカラムが必要です。

## よくある質問

**Q: 見出し番号がMarkdownファイルに書き込まれてしまう？**
A: 書き込まれません。番号は表示専用です。他のエディタで開くと番号は表示されません。

**Q: プラグインを有効にしたのに図・表の番号が表示されない？**
A: Obsidianの仕様により、プラグイン起動時にすでに開いているファイルには番号が自動で付与されません。以下のいずれかで反映されます：
- ファイルを一度閉じて開き直す
- コマンドパレット（`Ctrl+P`）から「Document Editor: 図表番号をリフレッシュ」を実行
- リボンの更新ボタン（🔄）をクリック

**Q: 図や表の番号がずれた？**
A: ページを再読み込み（`Ctrl+R` / `Cmd+R`）すると番号が正しく振り直されます。

**Q: 画像をペーストしたら自動変換されてしまう。wiki形式を使い続けたい。**
A: 設定の「wiki画像リンクを自動変換」をOFFにしてください。

**Q: ラベルに使える文字は？**
A: 英数字、ハイフン（`-`）、アンダースコア（`_`）が使えます。スペースは使えません。

**Q: 参考文献テーブルはどこに置けばいい？**
A: 文書のどこでもOKです。一般的には文書末尾に置くことを推奨します。

**Q: PDF出力でMathJaxの数式が表示されない？**
A: 数式のレンダリングが完了するまで少し待ってから出力してみてください。
