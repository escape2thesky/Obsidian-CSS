# REQUIREMENTS.md - Document Editor Plugin 機能要件

## プロジェクト概要
ObsidianでTeXライクな文書作成を支援するプラグイン。
技術論文・報告書の作成を想定。Obsidian Desktop専用。

---

## 機能要件チェックリスト

### Task 1: プロジェクト初期化
- [x] `document-editor/` サブディレクトリにプラグインを配置
- [x] `manifest.json` (id: document-editor, isDesktopOnly: true)
- [x] `package.json` + esbuild ビルド環境
- [x] `tsconfig.json` (strict mode)
- [x] `src/main.ts` - Plugin クラス
- [x] `src/settings.ts` - 設定型定義
- [x] `src/settingsTab.ts` - 設定UI（骨格）
- [x] `styles.css` - プラグイン全体のスタイル
- [x] `npm run build` が成功する

### Task 2: 見出し自動番号付け（表示のみ）
- [ ] H1 → `1.`, H2 → `1.1`, H3 → `1.1.1` 形式
- [ ] H4〜H6も対応（`1.1.1.1`形式）
- [ ] Markdownファイルの内容は変更しない（表示のみ）
- [ ] Live Preview (CodeMirror 6) で番号を表示
- [ ] Reading View (post-processor) で番号を表示
- [ ] 設定でON/OFF切替可能
- [ ] debounce 300ms でパフォーマンスを確保

### Task 3: 図・表の中央揃え & 番号自動付与
- [x] 画像（`![alt](src)`）を自動で中央揃え
- [x] 画像に「図N: キャプション」を自動で下に表示
- [x] キャプションはalt textから取得
- [x] テーブルを自動で中央揃え
- [x] テーブルに「表N」ラベルを表示
- [x] 削除時は番号が自動復活しない（DOM操作のみ）
- [x] 設定で図番号・表番号それぞれON/OFF切替可能
- [x] wiki画像リンク（`![[...]]`）を標準Markdown形式（`![](...)`)に自動変換
- [x] **起動時の初期表示なし（仕様）**: プラグイン起動時にすでに開いているビューには番号が自動付与されない。ファイルを開き直すか「図表番号をリフレッシュ」コマンドを実行することで反映される。

### Task 4: 図表ラベル & 相互参照
- [x] `{#fig:label}` 構文で図にラベルを付けられる
- [x] `{#tbl:label}` 構文で表にラベルを付けられる
- [x] `{#sec:label}` 構文で見出しにラベルを付けられる
- [x] `[ref:fig:label]` で図番号を参照（クリックでジャンプ）
- [x] `[ref:tbl:label]` で表番号を参照
- [x] `[ref:sec:label]` で見出し番号を参照
- [x] 存在しないラベルは `[???]` で表示
- [x] ラベルを含む構文文字はRendering Viewで非表示

### Task 5: 数式自動番号付け
- [x] ブロック数式 `$$...$$` に自動で `(1)`, `(2)` ... と番号を付ける
- [x] 番号は数式の右端に表示
- [x] `{#eq:label}` 構文でラベルを付けられる（数式直後の行に記載）
- [x] `[ref:eq:label]` で数式番号を参照
- [x] 設定でON/OFF切替可能

### Task 6: TOC生成コマンド
- [x] コマンドパレットから「TOCを挿入」コマンドを実行できる
- [x] H1〜H3の見出しを取得して目次を生成
- [x] 見出し番号（Task 2と同じ番号）を含む
- [x] カーソル位置に挿入、またはファイル先頭に挿入（コマンド2種）
- [x] 既存TOCブロックの更新も可能

### Task 7: 参考文献テーブル管理
- [x] `key` カラムを持つMarkdownテーブルを参考文献として認識
- [x] 参考文献テーブルに自動で番号を付与
- [x] 番号順: 参照順（設定）またはテーブル行順（設定）で切替
- [x] `[ref:bib:key]` → `[N]`（番号）に変換して表示
- [x] `[N]` クリックで参考文献テーブルの該当行にジャンプ
- [x] 存在しないキーは `[?]` で表示
- [x] 設定でON/OFF切替可能

### Task 8: PDF出力（Electron + フォント設定プレビュー）
- [x] コマンドパレットまたはリボンボタンから「PDFを出力」を実行できる
- [x] 出力前に設定ダイアログを表示
  - [x] 本文フォントサイズ（9〜14pt）
  - [ ] 見出しフォントサイズ比率
  - [x] 余白（mm）
  - [x] 出力ファイル名
- [x] ダイアログ内にA4サイズプレビューを表示
- [x] フォントサイズ変更時にプレビューをリアルタイム更新（debounce 500ms）
- [x] 見出し番号・図表番号・数式番号・参考文献番号がPDFに反映される
- [x] A4サイズ（210mm × 297mm）でPDF生成
- [x] Electron の `printToPDF` API を使用（Desktop専用）
- [x] 生成したPDFをVaultフォルダに保存

### Task 9: 設定画面 & 統合
- [x] Obsidian設定画面に「Document Editor」タブを追加
- [x] 全機能のON/OFFトグル
- [x] 参考文献番号順の切替（参照順 / テーブル行順）
- [x] PDFデフォルトフォントサイズ設定
- [x] PDFデフォルト余白設定
- [x] 設定変更が即座に反映される

## パフォーマンス要件

| 機能 | 上限 | 対策 |
|------|------|------|
| 見出し番号更新 | 300ms以内 | debounce 300ms |
| PDFプレビュー更新 | 1000ms以内 | debounce 500ms |
| post-processor全体 | 200ms以内/ページ | 軽量DOM操作のみ |
| PDF生成 | 30秒以内 | プログレス表示 |

## 技術スタック

| 項目 | 技術 |
|------|------|
| 言語 | TypeScript 4.7+ (strict mode) |
| ビルド | esbuild |
| フレームワーク | Obsidian Plugin API 1.0+ |

## 受け入れ条件

各Taskの受け入れ条件は、そのTaskを完了とみなす最低限の基準：

1. **Task 1**: `npm run build` が通り、Obsidianにプラグインが表示される
2. **Task 2**: サンプル文書でH1〜H3の番号が正しく表示され、ファイルが変更されない
3. **Task 3**: 画像貼り付けで図番号が表示され、テーブルが中央揃えになる（※起動時は番号が表示されない場合がある。これは仕様）
4. **Task 4**: `{#fig:label}` と `[ref:fig:label]` が正しく動作する
5. **Task 5**: ブロック数式に `(1)` が表示され、`[ref:eq:label]` が動作する
6. **Task 6**: コマンド実行でTOCが生成され、見出し番号と一致する
7. **Task 7**: 参考文献テーブルが認識され、`[ref:bib:key]` が `[1]` に変換される
8. **Task 8**: ダイアログが開き、プレビューが表示され、PDFが生成される
9. **Task 9**: 設定変更が保存・反映される
